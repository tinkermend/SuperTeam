## 架构

Control Plane:
PostgreSQL 任务队列
Redis/WebSocket 唤醒
Artifact Store
Human Decision Center

Runtime Agent:
Go daemon
HTTP claim + lease
WebSocket wakeup
本机 slot scheduler
Provider adapters
日志/事件回传

Provider:
Claude Code / OpenCode / Codex / Pi 子进程或 HTTP session

---

自组织, 固定组织

Agent 与 Agent 之间不要靠自由聊天，而是通过“工件 + 结构化结论 + 交接包”通信；中间由 Coordinator Agent 负责仲裁、路由和判断是否需要人类审批。
这样系统才可控、可审计、可暂停、可恢复，也符合真实团队协作方式。
